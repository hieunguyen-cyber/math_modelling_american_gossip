# Technical Report — Filemap and Run Order

This technical document describes the purpose and function of the main files and modules in the repository, a recommended execution order (CLI-friendly), where outputs are written, and how to interpret them. Use this as the canonical guide when reproducing experiments or extending the project.

## Recommended quick run (CLI)

1. Create virtualenv and install dependencies:

   python -m venv .venv
   source .venv/bin/activate
   python -m pip install -r requirements.txt

2. Run the full automated pipeline (experiments, figures, dashboard):

   bash scripts/run_all.sh

3. View built dashboard: `outputs/dashboard/index.html` or development copy `web/index.html`.

4. Export animation from the dashboard data (optional):

   python scripts/export_animation.py --source web/assets/data.json --out outputs/animations/propagation.gif


## High-level outputs and meaning

- `outputs/processed/` — CSV time-series results produced by experiment sweeps (e.g., `probability_sweep_ba.csv`). Columns: `q`, `f_mean`, `f_std`, `tau_mean`, `tau_std`.
- `outputs/figures/` — Static PNG/SVG figures generated by experiment scripts (e.g., `tau_vs_k_fit.png`, `tau_distribution.png`). These are publication-ready images summarizing fits and distributions.
- `outputs/dashboard/` — Built static dashboard (copy of `web/`) produced by `scripts/build_dashboard.py`. Open `index.html` to interact with precomputed assets.
- `outputs/animations/` — GIF/MP4 animations exported by `scripts/export_animation.py`.
- `server.log` — If `scripts/run_all.sh` started the Flask server, runtime logs are written here.


## File-by-file description

Top-level scripts
- `scripts/run_all.sh` — Full pipeline wrapper. Installs deps (pip), runs tests, runs experiments, generates figures, starts a temporary Flask server to generate live dashboard JSON, builds the dashboard, and stops the server. Use for end-to-end reproduction.
- `scripts/run_experiments.sh` — Run experiment suites only (sweeps and fits).
- `scripts/generate_figures.sh` — Generate figures from processed results.
- `scripts/build_dashboard.sh` — Build dashboard assets (calls `scripts/build_dashboard.py`).
- `scripts/run_tests.sh` — Run unit tests via pytest.

Server & streaming
- `scripts/server.py` — Lightweight Flask API providing `/api/generate` (POST). Accepts JSON: `graph_type`, `n`, `p`, `q`, `mode`, `k`, `attempt`. Returns JSON with `nodes`, `edges`, `history` and `meta` {victim, origin, graph_type, n} used by the frontend for live generation.

Animation exporter
- `scripts/export_animation.py` — Reads a `data.json` with graph + history and renders frames using NetworkX + matplotlib, then stitches into GIF/MP4 via `imageio`.

Graphs package (`graphs/`)
- `graphs/ba.py` — `generate_ba(n,m,seed)` wrapper for NetworkX BA graph.
- `graphs/ws.py` — `generate_ws(n,k,p,seed)` wrapper for NetworkX Watts–Strogatz.
- `graphs/erdos_renyi.py` — `generate_er(n,p,seed)` wrapper for NetworkX ER model.
- `graphs/apollonian.py` — Custom Apollonian generator (recursive triangle subdivision) implemented from scratch.
- `graphs/loaders.py` — Loaders for edge lists, adjacency lists, and CSV edge files.
- `graphs/utils.py` — Utilities (e.g., `largest_component`).

Propagation package (`propagation/`)
- `propagation/base.py` — `GossipPropagationEngine` core implementation. Constrained BFS on `G[N(v)]`, records `history` (list of frontiers), `timesteps`, and supports replay.
- `propagation/probabilistic.py` — `ProbabilisticGossip` subclass supporting probabilistic infection with `q`, and `one_shot` vs `repeated` attempt modes.
- `propagation/kth_neighbor.py` — `induced_k_hop_subgraph(G, center, k)` helper.
- `propagation/metrics.py` — `spread_factor(reached, degree)` and `spreading_time(history)`.

Experiments (`experiments/`)
- `reproduce_epl.py` — Simple driver to reproduce minimal EPL results (BA baseline example).
- `reproduce_pre.py` — Simple PRE driver for probabilistic baseline on ER graphs.
- `probability_experiments.py` — Parameter sweep over `q` for BA/ER/WS; produces CSVs under `outputs/processed/`.
- `scaling_laws.py` — Samples `tau` vs degree `k`, performs logarithmic fit `tau = A + B log(k)`, and writes `outputs/figures/tau_vs_k_fit.png`.
- `distributions.py` — Collects distribution of `tau` values, fits exponential, and writes `outputs/figures/tau_distribution.png`.

Visualization (`visualization/`)
- `static_plots.py` — Matplotlib utilities for publication-grade plots.
- `graph_renderer.py` — `render_pyvis` convenience for generating interactive HTML via `pyvis`.

Web dashboard (`web/`)
- `index.html` — Interactive frontend (Tailwind + vis-network + Plotly). Controls are bound to call `/api/generate` when available; otherwise loads `web/assets/data.json`.
- `assets/data.json` — Example/precomputed dataset used as fallback by the frontend. `scripts/build_dashboard.py` overwrites this when exporting demo graphs.

Data & outputs
- `data/raw/` — (empty) place raw empirical networks here (edge lists, CSVs).
- `data/processed/` — (optional) intermediate processed datasets.
- `outputs/processed/` — Experiment CSV outputs (sweeps and aggregates).
- `outputs/figures/` — Generated static PNG/SVG figures.
- `outputs/dashboard/` — Built dashboard copy (static) after `scripts/build_dashboard.py`.
- `outputs/animations/` — GIF/MP4 animations created by `scripts/export_animation.py`.


## Detailed recommended execution order and notes

1. `python -m pip install -r requirements.txt` — Ensure dependencies. Prefer running in a virtualenv.
2. `bash scripts/run_tests.sh` — Quick sanity checks.
3. `bash scripts/run_experiments.sh` — Run experiment suites. Output CSVs will be placed in `outputs/processed/`. Each CSV uses descriptive column names; `probability_experiments.py` produces `probability_sweep_{graph}.csv` containing mean/std of `f` and `tau` per `q`.
4. `bash scripts/generate_figures.sh` — Plot figures from `outputs/processed/` into `outputs/figures/`.
5. `python scripts/export_animation.py --source web/assets/data.json --out outputs/animations/propagation.gif` — Export a GIF of the front-end demo propagation, or point `--source` to any generated JSON with `nodes`, `edges`, and `history`.
6. `bash scripts/build_dashboard.sh` — Build a static copy of the dashboard to `outputs/dashboard/`.

If you prefer the end-to-end wrapper, `bash scripts/run_all.sh` does the above automatically and saves everything under `outputs/`.


## How to interpret common outputs

- `probability_sweep_ba.csv`: each row `q` indicates the average spread factor `f_mean` (fraction of victim neighbors reached) and `tau_mean` (average spreading time), with `*_std` the standard deviation across repetitions.
- `tau_vs_k_fit.png`: scatter of average `tau(k)` per degree bin and the fitted curve `A + B log(k)`. Use `scipy` fit outputs printed to STDOUT by `scaling_laws.py` for coefficients.
- `tau_distribution.png`: empirical histogram of `tau` with exponential fit overlay; fit parameter `lambda` interprets the decay rate P(tau) ~ exp(-lambda * tau).
- `outputs/animations/propagation.gif`: visual replay of the propagation; node color indicates infected vs susceptible; frame index matches timeline x-axis.


## Reproducibility notes

- Random seeds: experiment scripts accept `seed` where provided; `scripts/run_all.sh` sets deterministic seeds for reproducibility in most experiments. For large sweeps, use fixed seeds and keep `repetitions` consistent across runs.
- Performance: For N &gt; 10k you may need more memory and to run experiments in parallel (use `joblib` or `multiprocessing` to parallelize independent runs). The Flask demo server is intended for small-to-moderate graphs (N ≤ couple thousand). For very large graphs, implement server-side streaming or paging of frames.

## Contact and extension points

- To add new network generators, add `graphs/<name>.py` with `generate_<name>(...)` and export in `graphs/__init__.py`.
- To add a new metric or propagation rule, extend `propagation/` (add a new class similar to `ProbabilisticGossip`) and add a driver in `experiments/`.
